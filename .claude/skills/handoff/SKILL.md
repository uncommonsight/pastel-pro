---
name: handoff
description: Write HANDOFF.md before closing a session. Use when work is wrapping up, context is getting large, or user says done/stopping/closing.
---

Create or overwrite HANDOFF.md in the project root with:

## Last file edited
[filename + what changed]

## Work in progress
[exact task, specific function/line if relevant]

## Decisions made
[any choices made this session and why]

## Next steps
[numbered, specific — file names, function names, not vague]

## Blockers / open questions
[anything unresolved]

Keep it under 400 words. Written for a Claude with zero context.
